# Параметризация команды USE

Начиная с версии DaJet HTTP Server 4.0.15, доступен функционал параметризации команды ```USE```.

Данная возможность является полезной в тех случаях, когда требуется выполнить один и тот же скрипт в разных базах данных. В более ранних версиях псевдоним базы данных определялся строковой константой, что заставляло выполнять копирование одного и того же скрипта для решения подобных задач.

Для выполнения запроса DaJet Script необходимо выполнять привязку метаданных к объектам конкретной базы данных. Контекст такой привязки определяется командой ```USE```. Таким образом в новой версии скрипты делятся на два типа по способу привязки метаданных: статические и динамические.

Статические скрипты используют строковую константу, что позволяет их подготовить для выполнения один раз и закэшировать. Это очень эффективно с точки зрения производительности при многократном выполнении таких скриптов.

В свою очередь динамические скрипты используют переменную, значение которой устанавливается из входных параметров скрипта. С одной стороны, это добавляет больше гибкости и универсальности использования, но, с другой стороны, сказывается на производительности, так как привязку метаданных приходится выполнять при каждом выполнении динамического скрипта - контект базы данных заранее не известен.

Парсер DaJet Script автоматически определяет тип скрипта по способу использования команды ```USE```. Для того, чтобы скрипт считался динамическим достаточно использовать хотя бы одну команду ```USE```, параметризованную переменной. Тип такой переменной должен быть ```string```. Динамическая команда ```USE``` может находиться в любом месте скрипта, в том числе быть вложенной в дургие команды ```USE```.

**Пример динамического скрипта**

```SQL
DECLARE @БазаДанных string -- Параметризация USE
DECLARE @КодТовара  string
PRIVATE @Таблица    array

USE @БазаДанных -- Параметризация USE

  SELECT Ссылка, Код, Наименование
    INTO @Таблица
    FROM Справочник.Номенклатура
   WHERE Код = @КодТовара

END

RETURN @Таблица
```

**Пример параметров динамического скрипта**

```JSON
{
  "БазаДанных": "MS_TEST",
  "КодТовара": "MS-001"
}
```

> **Важно!** Передача параметров для команды ```USE``` динамического скрипта обязательна! Иначе будет выдана ошибка.

В случае если в скрипт будет передано значение синонима базы данных, который не зарегистрован на сервере, будет выдана ошибка следующего вида:

```
[2026-07-17 16:33:08] [API][ERROR][parametrized-use.djs] Metadata provider [MS_TEST] is not found.
```
