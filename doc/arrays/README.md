# Массивы значений

Начиная с версии DaJet HTTP Server 4.0.12, добавлена поддержка массивов. В основном эта функциональность появилась для использования массивов простых значений в запросах.

### Объявление массива

Массив, например строковых значений, объявляется следующим образом:

```SQL
DECLARE @Массив array(string)
```

### Инициализация массива

Массив можно заполнить значениями четырьмя способами.

1. Инициализация массива при объявлении.

```SQL
DECLARE @Массив array(string) = ['Код1', 'Код2', 'Код3']
```

2. Инициализация массива при помощи оператора присваивания.

```SQL
DECLARE @Массив array(string)

SET @Массив = ['Код1', 'Код2', 'Код3']
```

3. Инициализация массива запросом

```SQL
DECLARE @Массив array(string)

USE 'MS_TEST'
   SELECT TOP 10 Код INTO @Массив FROM Справочник.Номенклатура
END
```

4. Инициализация массива как входящего параметра для скрипта

В скрипте должна быть объявлена переменная типа ```array```.

```SQL
DECLARE @Массив array(string)
```

Параметры передаются в скрипт в теле HTTP-запроса.

```SQL
{
   "Массив": ['Код1', 'Код2', 'Код3']
}
```

### Использование массива в запросе

Массив значений может быть использован в запросах SELECT в операторах сравнения ```IN```.

```SQL
DECLARE @Массив array(string) = ['Код1', 'Код2', 'Код3']

PRIVATE @Таблица array

USE 'MS_TEST'
   SELECT Ссылка, Код, Наименование INTO @Таблица
     FROM Справочник.Номенклатура
    WHERE Код IN (@Массив)
END

RETURN @Таблица
```

### Получение массива в результат запроса

Массив может быть результатом запроса.

```SQL
PRIVATE @Массив array(entity) -- Массив ссылок

USE 'MS_TEST'
   SELECT TOP 10 Ссылка INTO @Массив
     FROM Справочник.Номенклатура
END

RETURN @Массив
```
